# Buro Visual Identity

## Brand Foundation

- **Name:** Buro
- **Meaning:** conversation
- **Planned production domain:** `buro.ss` (not registered yet)
- **Tagline:** Start jazzing.
- **Purpose:** live audio rooms for jokes, stories, laughter, and casual conversation

Buro should feel casual, warm, quick, and alive. The product should invite people into conversation without looking like a heavy social platform.

## Brand Personality

Buro is:

- Conversational.
- Playful.
- Human.
- Lightweight.
- Audio-first.
- Spontaneous.
- Welcoming.

Buro is not:

- Corporate.
- Creator-economy heavy.
- Finance-like.
- Video-first.
- Overproduced.
- Feature-cluttered.

## Voice And Tone

The product voice should be short, relaxed, and friendly.

Preferred language:

- Start jazzing.
- Join the room.
- Leave quietly.
- You are muted.
- You are live.
- Random Jazz.
- Open Mic.

Avoid:

- Complex creator terminology.
- Marketplace language.
- Follower-growth language.
- Monetization prompts.
- Formal conference wording.

## MVP Copy Guidelines

### Primary CTA

Use:

- Start jazzing
- Join room
- Start a room

Avoid:

- Create broadcast
- Launch event
- Start livestream
- Go on stage

### Empty State

Example:

```text
No rooms live right now.
Start jazzing and bring people in.
```

### Room Creation

Room title and category are optional. The UI should make this clear.

Example:

```text
Name the room if you want.
Blank is fine.
```

### Mute State

Example:

```text
You joined muted.
Unmute whenever you're ready.
```

## Visual Direction

Buro should use a mobile-first interface with large touch targets and clear audio controls.

Recommended design qualities:

- Warm dark or soft light base.
- Strong accent color for live and active states.
- Rounded cards.
- Clear room cards.
- Minimal navigation.
- Prominent mute/unmute button.
- Chat and reactions visible but secondary to audio.

## Color Direction

**No gradients. Use flat solid colors only.**

Buro's UI must use flat, solid colors. Do not use CSS gradients, `linear-gradient`,
`radial-gradient`, gradient-like backgrounds, gradient buttons, gradient cards, or
gradient decorative effects. Borders, shadows, spacing, opacity, and subtle
solid-color surfaces are allowed.

Approved solid palette:

- Warm Orange `#F59E0B`
- Deep Charcoal `#171717`
- Earth Brown `#7A4A1D`
- Soft Cream `#FFF7E8`
- Live Green `#22C55E`
- Danger Red `#EF4444`

The palette should support:

- A distinctive primary brand color (Warm Orange).
- A clear live indicator color (Live Green).
- A calm background (Deep Charcoal).
- High contrast text (Soft Cream).
- Accessible disabled and muted states (solid colors with opacity).

Usage direction:

- Deep charcoal for audio-room atmosphere.
- Warm orange accent for jazz energy and primary actions.
- Soft cream for text surfaces.
- Green for live status.
- Keep surfaces flat and disciplined — depth comes from borders and shadows, never gradients.

## Typography Direction

Typography should feel modern and readable.

Guidelines:

- Use one primary sans-serif family.
- Prioritize legibility on mobile.
- Keep room titles scannable.
- Avoid decorative fonts for functional UI.
- Use playful copy, not hard-to-read typography, for personality.

## Logo Direction

The MVP does not require a complex logo system. A simple wordmark is enough.

Possible directions:

- `Buro` wordmark with rounded letterforms.
- Small sound-wave or speech-inspired mark.
- Simple dot or pulse motif for live audio.

Avoid:

- Microphone icons that feel generic.
- Podcast branding.
- Video camera symbols.
- Heavy broadcast or stage imagery.

## UI Priority

The room screen is the most important MVP surface.

Hierarchy:

1. Room title and live state.
2. Participant/audio state.
3. Mute/unmute control.
4. Leave room control.
5. Chat.
6. Reactions.
7. Host moderation controls.

## Accessibility

Minimum expectations:

- Clear focus states.
- Keyboard-accessible controls where applicable.
- Sufficient text contrast.
- Touch targets suitable for mobile.
- Mute/unmute state available as text, not color alone.
- Reactions should not be required to understand the conversation.

## Scope Guardrails

Do not design MVP screens for:

- Video rooms.
- Recorded replays.
- Feeds.
- User follower profiles.
- Private inbox.
- Gifts.
- Wallets.
- Stories.
- Creator dashboards.

The visual identity should help people enter live conversations quickly, not make Buro feel like a large social network before the core room loop works.
